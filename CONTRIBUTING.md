# Contributing

Keep the skill narrow and reliable.

## Good additions

- clearer expression examples
- real n8n failure modes
- new workflow patterns that recur often
- notes about node quirks that actually break workflows

## Avoid

- generic AI workflow fluff
- claims that cannot be verified in n8n docs
- giant one-off scripts unless they solve repeated pain
- mixing unrelated platform guidance into the n8n skill

## Rule

If a new note does not help an agent choose a better n8n workflow, node configuration, or validation step, leave it out.
